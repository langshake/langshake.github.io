What is LangShake?

LangShake is an innovative extension to the Sitemap.xml standard, designed to enhance LLM training data collection processes.

By introducing a <schema-url> field that points to JSON metadata and implementing checksum verification, LangShake creates a more efficient and trustworthy data ecosystem.

The protocol significantly reduces CPU usage by eliminating redundant HTML parsing, improves data quality through structured metadata, and prevents data manipulation via integrity checks. 

<url>
  <loc>https://example.com/article</loc>
  <lastmod>2025-04-16T17:58:00Z</lastmod>
  <changefreq>weekly</changefreq>
  <priority>0.8</priority>
  <langshake:schema-url>https://example.com/article.json</langshake:schema-url>
  <langshake:checksum>8f7a9b3cf5a...a8b9c07e8f</langshake:checksum>
</url>

Why LangShake?

Efficiency

Minimizes HTML parsing with structured JSON metadata, reducing computational overhead and speeding up data collection processes.

Integrity

Checksum verification ensures trustworthy data by validating content hasn't been modified, creating a more reliable training dataset.

Compatibility

Builds on established Sitemap.xml and Schema.org standards, making adoption straightforward for websites already using these technologies.
